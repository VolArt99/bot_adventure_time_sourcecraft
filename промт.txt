1. Роль и задача
Ты — опытный Python-разработчик, специализирующийся на создании Telegram-ботов. 
Твоя задача — разработать:
•	полный рабочий код бота
•	архитектуру проекта
•	инструкцию по запуску
для Telegram-бота управления мероприятиями в частной группе с включёнными темами (Forum Topics).

Цель группы: Объединение людей для активного досуга, общения и совместных мероприятий.
Размер группы: ~219 участников

Бот должен:
•	помогать создавать мероприятия
•	управлять участниками
•	отправлять напоминания
•	предоставлять актуальную информацию по событиям
•	минимизировать участие администратора

2. Ограничения и контекст работы
•	Бот должен быть максимально бесплатным
•	Работает только в частной группе и в личных сообщениях с участниками
•	Поддерживает Telegram Topics (threads) — бот должен быть администратором для доступа к `message_thread_id`
•	Минимальная нагрузка на администратора
•	Удобный UX (inline-кнопки)
•	Хранение данных: YDB (Yandex Database)
•	Дружелюбный краткий стиль, понятные сообщения, актуальные данные, минимальный спам.

3. Технический стек
•	Язык: Python 3.+.
•	Библиотека: aiogram 3.x (асинхронная).
•	База данных: YDB (Yandex Database)
•	Планировщик: APScheduler (хранилище SQLite или in-memory)
•	Внешние API: только бесплатные (OpenWeatherMap для погоды).
•	Хостинг: бесплатный с поддержкой фоновых задач (Render, Railway, PythonAnywhere)
•	Работа с темами: использование `get_forum_topics`, отправка с параметром `message_thread_id`
•	Устойчивость: восстановление активных мероприятий и перепланирование напоминаний при старте

4. Основной функционал
4.1 Создание мероприятия
- Доступно только организаторам (настраиваемый список пользователей).
- Участник запускает команду `/create_event` в ЛС с ботом.
- Бот последовательно запрашивает:
•	название
•	описание (опционально)
•	дата и время
•	длительность (опционально)
•	место проведения (текст + геолокация)
•	стоимость (общая и на человека)
•	лимит участников
•	нужна ли машина (Да/Нет) — для карпулинга
•	тема (ветка) для публикации
•	организатор
•	категория (спорт, прогулки, поездки, игры, культура, еда, обучение). 
- Погода: автоматически проверяется через OpenWeatherMap (иконка + описание) и добавляется в анонс.
- После сбора данных бот публикует форматированное сообщение в выбранной теме по шаблону:
[Название] [Описание] 
Погода: [Иконка погоды]
🗓 [Дата] в [Время] на [Длительность]
📍 [Адрес/Локация]
💰 [Стоимость]
Кто уже идет [X]/[Лимит]:
[Список @юзернеймов]
Резерв (если найдем еще людей):
[Список @юзернеймов]”
- Inline-кнопки: «Пойду», «Отказаться», «В резерв».
- При включённой опции «Нужна машина» добавляются кнопки «Еду на машине» (уточнить места) и «Ищу попутку» (выбор водителя).
- Под сообщением динамически обновляются списки участников.

4.2. Управление участием:
- «Пойду» — добавляет в основной список, если есть места. При достижении лимита кнопка блокируется.
- «В резерв» — добавляет в резерв (если пользователь был в основном, он освобождает место, и первый из резерва автоматически переходит в основной с уведомлением).
- «Отказаться» — удаляет из основного или резерва. При освобождении места бот предлагает его первому из резерва (ЛС: "Освободилось место на [Ивент], подтвердить?").
- Запрет на одновременное нахождение в основном и резервном списках.
- При любом изменении бот редактирует исходное сообщение, обновляя списки и счётчик участников.
- Если включён карпулинг, под основным сообщением отображается список водителей и пассажиров:
"Водители: 
@user1 [X]/[Лимит]: 
[Список @юзернеймов]
@user2 [X]/[Лимит]: 
[Список @юзернеймов]".

4.3. Финансы (авто-расчет):
- В сообщении автоматически пересчитывается стоимость с человека: общая стоимость / количество участников (если участников 0 — показывается полная стоимость).

4.4. Напоминания о мероприятии
- За настраиваемые интервалы (1 день, 3 ч, 2 ч, 1 ч, 30 мин) бот отправляет:
  - личные сообщения участникам основного списка (и опционально резерва) с названием, датой, временем, местом и ссылкой на сообщение в группе.
  - сообщение в тему группы с напоминанием.
- Для отправки ЛС пользователь должен инициализировать диалог командой `/start` (это оговаривается в правилах).

4.5. Дайджест мероприятий
- Раз в неделю (настраиваемо) бот формирует и публикует дайджест в отдельную тему «Календарь Тусовок».
- Формат:
"📅 Афиша на [Даты недели]
🔥 [Название]
🗺 Где: [Место]
🗓 Когда: [Дата и Время]
По всем вопросам @[юзернейм_организатора]
⛓ Подробнее о мероприятии в ветке "[Название темы]""
- Для каждого мероприятия — ссылка на сообщение (или тему) в группе, контакт организатора, описание.
- Команда `/digest` для ручного формирования за период (неделя/месяц).
- Завершённые мероприятия (по дате) не участвуют в дайджесте.

6. Архитектурные и эксплуатационные требования
6.1. Структура базы данных (YDB)
- users: id, username, notification_settings, stats_count, birth_date (опционально)
- events: id, title, date_time, duration, location, price, limit, thread_id, message_id, creator_id, weather_info, carpool_enabled, status (active/completed/cancelled)
- participants: event_id, user_id, status (going/waitlist/driver/passenger)
- reviews: event_id, user_id, rating, comment
- waitlist_queue (опционально) — для управления очередностью резерва

6.2. Middleware и безопасность
- Бот игнорирует сообщения от пользователей, не состоящих в основной группе.
- Для команд в ЛС проверяется, является ли пользователь участником группы.

6.3. Работа с темами
- При создании мероприятия бот получает список тем группы через `get_forum_topics` и предлагает выбрать.
- Сообщение публикуется с указанием `message_thread_id`.
- В дайджесте используются ссылки вида `t.me/c/chat_id/message_id` для прямого перехода.

6.4. Стабильность и отказоустойчивость
- При старте бот восстанавливает активные мероприятия из БД и перепланирует напоминания.
- Обработка ошибок (например, API погоды не отвечает — пропустить погоду, но не падать).
- Автоматическое создание файла БД и таблиц при первом запуске.
- Логирование ошибок в файл или отдельную тему.

6.5. Код и структура проекта
- Рекомендуемая структура:
bot/
├── main.py # точка входа, регистрация роутеров, запуск
├── config.py # переменные окружения (токен, API ключи)
├── database.py # работа с БД (YDB)
├── keyboards.py # клавиатуры (inline, reply)
├── handlers/
│ ├── init.py
│ ├── events.py # создание, редактирование, просмотр
│ ├── participation.py # кнопки Пойду/Отказаться и т.д.
│ ├── digest.py # дайджест
│ ├── reminders.py # напоминания
│ └── admin.py # админ-функции
├── utils/
│ ├── weather.py # интеграция с OpenWeatherMap
│ └── scheduler.py # настройка APScheduler
└── texts.py # константы сообщений
- Использовать `Router` из aiogram для группировки хендлеров.

7. Инструкция по развёртыванию (free tier)
1. Регистрация бота 
 - Написать @BotFather, создать бота, получить токен.  
 - Отключить privacy mode (разрешить боту читать все сообщения).  
 - Установить команды через `/setcommands`.

2. Добавление в группу 
 - Добавить бота в группу, сделать администратором с правами:  
   - отправка сообщений в темы  
   - редактирование сообщений  
   - удаление сообщений (опционально)  
   - приглашение пользователей (не обязательно)  

3. Получение API ключа погоды 
 - Зарегистрироваться на [OpenWeatherMap](https://openweathermap.org/), взять бесплатный API ключ.

4. Подготовка кода 
 - Клонировать репозиторий, создать файл `.env` с переменными:  
BOT_TOKEN=...
WEATHER_API_KEY=...
GROUP_ID=... # числовой ID группы
YDB_ENDPOINT=grpcs://ydb.serverless.yandexcloud.net:2135
YDB_DATABASE=/ru-central1/b1gburmf2sv6jdb39qi4/etnnggcnvg783q3fj957
# Для аутентификации в YDB необходимо установить одну из следующих переменных:
# YDB_SERVICE_ACCOUNT_KEY_FILE_CREDENTIALS - путь к файлу ключа сервисного аккаунта
# YDB_ACCESS_TOKEN_CREDENTIALS - OAuth-токен
# YDB_ANONYMOUS_CREDENTIALS=1 - для анонимного доступа (если разрешено)
# YDB_METADATA_CREDENTIALS=1 - для использования метаданных ВМ
- Установить зависимости: `aiogram`, `apscheduler`, `python-dotenv`, `aiohttp`, `ydb`, `ydb[aio]`.

5. Развёртывание на бесплатном хостинге
- Render:  
- Создать веб-сервис из репозитория, указать команду запуска `python main.py`.  
- Для фоновых задач настроить cron job (отдельный файл-триггер или использовать APScheduler внутри).  
- Для работы с YDB необходимо настроить сервисный аккаунт и подключение.  
- PythonAnywhere:  
- Загрузить код, создать виртуальное окружение.  
- Использовать Always-on task (платно) или настроить cron для дайджеста, а напоминания реализовать через отдельный скрипт, запускаемый по расписанию.  
- YDB предоставляет managed базу данных в облаке.  
- Railway:  
- Поддерживает подключение к YDB через сервисный аккаунт.

6. Проверка работоспособности  
- Запустить бота, убедиться, что он отвечает на `/start`.  
- В группе проверить создание мероприятия, запись, напоминания.  
- При необходимости настроить часовой пояс в APScheduler.
