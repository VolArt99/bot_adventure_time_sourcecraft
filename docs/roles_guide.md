# Инструкция по возможностям бота по ролям

## Владелец (OWNER_ID)

**Доступ:** без ограничений по командам и лимитам.

### Что может делать
- Полный набор пользовательских и админских команд.
- Одобрять/отклонять заявки в онбординге.
- Контролировать «Рассказ о себе»:
  - `/pending_intro` — кто не завершил;
  - `/list_intro` — статусы и дедлайны.
- Запускать сервисные проверки:
  - `/debug_info`, `/health`, `/status`.

## Админ (ADMIN_IDS)

**Доступ:** все команды, но с дневным лимитом `ADMIN_DAILY_COMMAND_LIMIT`.

### Основные команды
- Управление событиями: `/create_event`, `/edit_event`, `/cancel_event`.
- Сервис и модерация: `/admin_report`, `/list_topics`, `/update_topic_names`, `/random_pairs`.
- Статистика и поиск: `/my_stats`, `/top`, `/find_events`.

## Участник группы (одобренный)

**Доступ:** только пользовательские команды, с дневным лимитом `MEMBER_DAILY_COMMAND_LIMIT`.

### Доступные команды
- `/start`, `/help`, `/status`
- `/create_event`, `/my_events`, `/digest`
- `/subscriptions`, `/my_digest`
- `/my_stats`, `/top`, `/find_events`
- `/random_optin`, `/random_optout`

## Не одобренный пользователь (до вступления)

**Доступ до одобрения:**
- `/start` — начать онбординг;
- `/help` — короткая справка;
- `/status` — проверка доступности бота.

Остальные команды блокируются до одобрения заявки владельцем.

## Важно про темы Telegram

Telegram Bot API не предоставляет надёжный универсальный способ «получить все ID тем» одной командой. Поэтому бот:
- автоматически сохраняет ID темы из входящих сообщений в темах;
- использует `topics_config.py` как источник человеко-читаемых названий.
